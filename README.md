# WBO Animation - генератор видео-карточек

Десктопное приложение для Windows на Python, которое:

- накладывает видео (режим наложения «Экран») на пачку изображений (карточки маркетплейса);
- позволяет для каждой карточки выбрать своё видео и настройку «кривых»;
- экспортирует готовые видео-карточки в формате `.mp4`.

## Требования

- Windows 10/11;
- установленный Python 3.10+ (для разработки/сборки);

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

На Windows можно запускать двойным кликом по `run.bat` или командой:

```bash
python main.py
```

## Сборка дистрибутива (для запуска без установленного Python)

Приложение собирается в формате **папки-дистрибутива** с exe и всеми ассетами.

1. Запустите скрипт сборки:

   ```bash
   build.bat
   ```

   При запуске скрипт спросит режим:

   - **[1] With console (debug)** — exe запускается с консолью; удобно видеть traceback и логи.
   - **[2] Without console (GUI only)** — обычный GUI‑режим без консоли (для пользователей).

   В обоих случаях `build.bat`:

   - установит/обновит зависимости из `requirements.txt` (через тот же Python, что и `run.bat`);
   - соберёт приложение через PyInstaller в режиме `onedir` (`dist\WBOAnimation\...`);
   - скопирует папку `Assets` рядом с exe;
   - создаст удобный лаунчер `run_here.bat` для запуска с консолью (видно логи/ошибки).

2. Готовый дистрибутив находится в папке:

   ```text
   dist\WBOAnimation\
   ```

   Внутри неё:
   - `WBOAnimation.exe` — основной исполняемый файл;
   - `Assets\` — все иконки, изображения, видео и звуки, необходимые приложению;
   - `run_here.bat` — запуск exe с открытой консолью (полезно при отладке).

3. Для передачи пользователю/запуска на другой машине скопируйте **всю папку** `dist\WBOAnimation\` и запускайте приложение только из неё.

## Проверка обновлений

В меню **Помощь → Проверить обновление** приложение сверяет свою версию с последним [релизом на GitHub](https://github.com/divangames/WbO-BUMP/releases). Если вышла новая версия, откроется диалог с кнопкой «Открыть страницу загрузки» — пользователь переходит на страницу релиза и может скачать установщик или архив.

## Публикация обновлений (для разработчика)

Чтобы пользователи видели новую версию в «Проверить обновление»:

1. Соберите дистрибутив: `build.bat` (режим без консоли для пользователей).
2. Соберите установщик (если используете): `build_installer.bat` — результат в `installer_output\`.
3. В [репозитории GitHub](https://github.com/divangames/WbO-BUMP) создайте **Release**:
   - **Tags**: выберите или создайте тег в формате версии, например `v0.1.2.3` (обязательно «v» в начале).
4. В описании релиза укажите изменения; в **Assets** загрузите файлы для скачивания:
   - установщик `WboBAMP_Setup_X.Y.Z.Z.exe`, и/или
   - архив папки `dist\WboBAMP` (или `dist\WBOAnimation`) для тех, кто ставит без установщика.
5. Чтобы в приложении работала кнопка **«Загрузить и применить»** (автообновление без ручной установки), приложите к релизу ZIP с обновлёнными файлами. Имя файла должно содержать слово **update** и расширение **.zip**, например: `WboBAMP_update_0.1.2.3.zip`. Внутри архива — содержимое папки приложения (например одна папка `WboBAMP` с файлами `Assets`, exe и т.д.). Программа скачает этот ZIP и перезапишет свои файлы (кроме запущенного exe).

Версия в коде (`APP_VERSION` в `main.py`) должна совпадать с тегом релиза (без «v»). После публикации релиза пункт «Проверить обновление» предложит пользователям перейти на страницу загрузки.

## Пользователи, токены и админ-панель

### Регистрация и вход

- В верхнем меню есть пункт **«Пользователь»**.
- Любой пользователь может:
  - зарегистрировать новый аккаунт (логин + пароль хранятся локально в `users.json` в виде хэша);
  - войти под своим логином/паролем;
  - просмотреть профиль (логин, роль, текущий баланс токенов и количество непрочитанных сообщений);
  - выйти из аккаунта.
- Без входа приложение работает в режиме **гостя**: токены не учитываются, экспорт не ограничен.

### Токены (кредиты за рендер)

- У каждого зарегистрированного пользователя есть баланс токенов.
- **Стоимость экспорта**:
  - 1 успешно отрендеренная видео‑карточка = **5 токенов**.
  - Для экспорта N карточек нужно минимум `N × 5` токенов на балансе.
- При запуске экспорта:
  - если токенов не хватает, экспорт не начинается и показывается предупреждение;
  - после завершения успешные карточки списывают токены, новый баланс сохраняется в `users.json` и отображается в статус‑баре.
- Внизу окна (строка статуса) всегда видно:
  - текущего пользователя и его роль;
  - текущий баланс токенов.

### Администратор и админ-панель

- При первом запуске (или если нет `users.json`) автоматически создаётся администратор:
  - логин: `Delbraun`
  - пароль: `0211Fenix91!`
- Вход под этим пользователем открывает доступ к **админ-панели**:
  - пункт «Админ-панель» появляется в меню **«Пользователь»**;
  - окно админ‑панели позволяет:
    - просматривать список пользователей (логин, роль, токены, статус);
    - изменять количество токенов у выбранного пользователя;
    - блокировать/разблокировать пользователей;
    - менять пароли пользователей;
    - отправлять текстовые сообщения конкретному пользователю (они показываются при входе и в профиле);
    - удалять пользователя (кроме текущего администратора `Delbraun`, с подтверждением).
- Заблокированные пользователи:
  - не могут войти в приложение;
  - если были заблокированы после входа, не могут запускать экспорт (получают сообщение «аккаунт заблокирован»).

## Структура ассетов

- `Assets/video` — сюда кладём видео для наложения (`.mp4`, до 8 секунд, 900x1200 px);
- `Assets/demo` — демонстрационные карточки (изображения), которые можно загрузить в приложение.
